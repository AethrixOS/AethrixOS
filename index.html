About

Not a distribution. A foundation.

Most operating systems people call "new" are Linux with a different wallpaper. Aethrix is not that. Every subsystem — the bootloader, the memory manager, the scheduler — is written from first principles, so each decision can be examined instead of inherited.

Built from scratch

No forked kernel, no borrowed init system. The architecture is Aethrix's own, designed to be read and understood.

Security by design

Isolation and least-privilege are decided at the architecture stage, not patched in after a release.

Documented in the open

Every subsystem ships with the reasoning behind it, so contributors can extend it without guessing.

Features

Engineering choices, not marketing bullets

Six commitments that shape every design decision inside Aethrix.

Performance

A scheduler and memory manager tuned for predictable latency, not just peak throughput.

Security

Sandboxed drivers and a minimal trusted computing base, enforced at the kernel boundary.

Open source

Every line of the kernel and userland tooling is public, versioned, and reviewable under MIT.

Modular

Subsystems communicate through defined interfaces, so a driver or filesystem can be swapped, not rewired.

Developer friendly

A documented syscall surface, native terminal tooling, and build scripts that don't fight you.

Modern architecture

Designed around today's hardware — multi-core scheduling and modern filesystem semantics from day one.
